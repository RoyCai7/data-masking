1
Show the raw file — 1 min
Open demo/support_ticket_export.txt in a text editor

Say: "This is a realistic support ticket export — the kind of file a support team might send to a third-party partner. Scroll through it slowly. Notice what's hiding in here."

Root passwords SSH private key AWS access key + secret Credit card number JWT token DB connection string with password Internal IP addresses Customer email addresses SUSE Reg codes FTP credentials

Say: "This file cannot be shared as-is. Manually redacting it is slow, error-prone, and impossible to audit."

2
Upload and mask via the Web UI — 2 min
Switch to browser tab — http://<server-address>

Say: "The web interface is what most users will see. No training needed — just drag and drop."

  1. Drag support_ticket_export.txt onto the upload area (or click to browse)
  2. Click Upload & Mask — watch the progress indicator
  3. When status shows Completed — click Download Masked File
  4. Open the downloaded file in your text editor — put the two windows side by side
Say: "Every credential, key, email, and IP is replaced with a clear label — [PASSWORD], [AWS_ACCESS_KEY], [EMAIL] — while all the other content stays intact. The file is readable, just safe to share."
3
Show the risk report — 1 min
Click the Report button on the completed task

Say: "Every masking job produces a full audit report. This is your compliance evidence — you can prove exactly what was found and masked."

  1. Point to the Risk Score — explain it reflects volume and sensitivity of findings
  2. Scroll through the per-rule breakdown — show match counts per category
  3. Click an example row — show the original value and its masked replacement with file + line reference

Say: "This report can be saved and attached to your change record or compliance ticket."

4
Show rule management — 2 min
Click Rules in the header nav

Say: "The system ships with built-in rules for common sensitive patterns — emails, passwords, API keys, IPs, credit cards. You can see exactly which rules are active, what they match, and toggle them on or off."

  1. Filter by a category — e.g. Credential
  2. Toggle one rule off, then back on — to show live control
  3. Open a rule detail — show the regex pattern, strategy, and weight

Say: "But every organisation has unique data. SUSE has registration codes in the format Regcode-XXXXX. That's not in any generic ruleset. Let me show you how to add that in 30 seconds."

5
✨ AI rule creation — the wow moment — 3 min
Switch to terminal

Say: "Writing a regex by hand takes expertise most people don't have. With our AI integration, you just describe what you want in plain English — the system does the rest."

# Describe the pattern in plain English python3 llm_add_rule.py \ -k $K \ -d "SUSE registration code in the format Regcode-XXXXX where X is uppercase letter or digit" \ -r "Needed for support ticket export masking"
  1. While it runs, say: "The local AI model reads the description, generates a regex, and tests it against examples — all on-premise. Nothing leaves the server."
  2. When output appears: "It generated the pattern, showed what it matches and what it doesn't, and submitted it for review — one command."
  3. Switch to the browser → Admin Console → Suggestions — the new suggestion is pending
  4. Click Approve
  5. Re-upload support_ticket_export.txt — the Regcode-ABCDE values are now masked
Say: "The suggestion workflow means anyone on the team can propose a rule — they don't need to know regex. A reviewer approves it before it goes live. No accidents."

🏁 Closing — 30 seconds

"The full cycle is: upload, mask, download, report — in under 30 seconds. Rules are centrally managed with role-based access. New patterns can be added by anyone via AI suggestion, reviewed and approved by admins. Everything is logged and auditable. And it all runs on-premise — your data never leaves your infrastructure."

❓ Likely Questions & Answers
QuestionAnswer
Does it work on archives / zip files?Yes — upload a .tar.gz or .zip; every text file inside is masked and the archive is repacked. The report shows which file and line each match came from.
Is data sent to the cloud?No — everything runs on-premise. The LLM model runs locally via Ollama. No data leaves your server.
Who controls which rules are active?Admins manage system-wide rules. Org owners manage org rules. Regular users can submit suggestions but cannot create rules directly.
How do we add custom patterns?Describe it in plain English with llm_add_rule.py, or write the regex manually in the Admin Console.
Can we integrate this into CI/CD?Yes — full REST API. The mask_api.sh script can be called from any pipeline.
What file types are supported?Any plain text file — logs, config files, exports. Also .tar.gz, .tgz, and .zip archives.
Can it handle large files?Yes — processing is asynchronous. Upload returns immediately; you poll for completion. The task queue handles concurrent uploads.
