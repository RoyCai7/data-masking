# 数据脱敏服务 (DMS) 全能解决方案

## 项目概况

| 项 | 说明 |
|---|------|
| **目标** | 通过自动化工具取代耗时的手动日志脱敏（30-60min → 5min），确保 100% 数据安全，防止 AI 时代的隐私泄露 |
| **周期** | 2026年2月23日 – 2027年2月22日（约52周） |
| **团队** | Roy Cai, Richard Fan|

---

## 1. 三模组架构 (3-Module Architecture)

项目通过三个独立通道满足不同办公场景：

| 模组 | 面向对象 | 功能描述 |
|------|---------|---------|
| **Web UI（文件上传）** | 普通用户 | 支持大文件（30MB）拖拽上传、实时进度显示及可视化风险报告 |
| **REST API（系统集成）** | 开发者 | 提供标准的 FastAPI 接口，支持 CI/CD 流水线自动脱敏及第三方工具集成 |
| **Tampermonkey 脚本（浏览器助手）** | 日常办公 | 实时监控输入/粘贴行为，自动拦截并脱敏 Bugzilla、ChatGPT 等页面的敏感信息 |

---

## 2. 技术栈 (Tech Stack)

- **后端**：Python 3.10 + FastAPI；多线程并发处理（16 Workers）；SQLite 规则库
- **前端**：React 18 + TypeScript + Tailwind CSS；Vite 构建
- **脱敏引擎**：自研正则匹配逻辑，支持 ASTERISK（星号）、HASH、部分隐藏等多种策略
- **部署**：Docker 容器化部署，支持 WebSocket 实时通信
