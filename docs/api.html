Data Masking Service

REST API for sensitive data detection and masking in files and archives.

Base URL  http://10.146.15.188:8080lt;server-address10.146.15.188:8080gt;/api/v1
Version  1.0
Format  JSON

🔑 Authentication

All endpoints (except GET /status and GET /rules) require an API key passed in the request header:

X-API-Key: dms_your_api_key_here

Session isolation is optional — pass X-Session-ID to keep tasks grouped. If omitted, a session is auto-created on first upload.

Quick Start
Mask a file in four steps:
📤
1. Upload
POST /mask
⏳
2. Poll
GET /task/{id}
📥
3. Download
GET /download/{id}
📊
4. Report
GET /report/{id}

Masking
Upload files for sensitive data detection and masking. Supports plain text, logs, and archives (.tgz, .tar.gz, .zip).
POST /mask Upload and mask a file Auth required ▼
any authenticated user
Asynchronous — returns a task_id immediately. Poll GET /task/{task_id} to check completion.
Request — multipart/form-data
FieldTypeRequiredDescription
filebinary✔ requiredFile to mask. Plain text or archive (.tgz, .tar.gz, .zip)
whiteliststringoptionalComma-separated values to skip masking, e.g. localhost,127.0.0.1
Headers
HeaderRequiredDescription
X-API-Key✔API key — determines which rules (private/org/system) are applied
X-Session-IDoptionalSession UUID for task grouping. Auto-created if omitted.
Response 200
{ "task_id": "fea215c0-eb5f-4715-b4df-cf8d8567fd63", "session_id": "d27c82de-574e-4326-ade8-073b2b562566", "status": "pending", "filename": "secrets.log", "message": "File uploaded successfully. Processing started." }
GET /task/{task_id} Get task status Auth required ▼
Path Parameters
ParamDescription
task_idUUID returned by POST /mask
Response 200 — status: pending | processing | completed | failed
{ "task_id": "fea215c0-...", "filename": "secrets.log", "status": "completed", "progress": 100, "created_at": 1748433173.95, "completed_at": 1748433174.12, "error": null, "report": { /* full report object when completed */ } }
GET /tasks List all tasks in current session Auth required ▼

Returns all tasks associated with the current X-Session-ID. Useful for resuming sessions.

GET /download/{task_id} Download the masked file Auth required ▼

Returns the masked file as a binary download. For archives, returns a repacked archive of the same type with all text files masked inside.

Use curl -OJ to save with the original filename.
GET /report/{task_id} Get detailed masking report Auth required ▼
Response 200
{ "report_id": "report-fea215c0-...", "generated_at": "2026-05-28T13:29:33Z", "file_info": { "name": "secrets.log", "size_bytes": 45231, "lines_total": 2261, "is_archive": false, "files_processed": 1 }, "summary": { "total_matches": 42, "risk_score": 67, "risk_level": "HIGH", "processing_time_ms": 320.5, "whitelist_skipped": 3 }, "breakdown": [ { "rule_id": "email_address", "rule_name": "Email Address", "matches": 14, "examples": [ { "line": 42, "file": "logs/app.log", "original": "user@example.com", "masked": "[EMAIL]" } ] } ] }
For archive files, each example includes a file field with the relative path inside the archive.
Rules
Masking rules define what patterns to detect and how to mask them. Rules have three scopes:
system — all users org — org members only private — creator only
GET /rules List rules visible to caller No auth needed ▼
public (system rules only without auth)
Query Parameters
ParamTypeDescription
categorystringFilter by category: PII, Credential, Network, Finance, Health, Custom…
enabled_onlybooleanReturn only enabled rules (default: false)
scopestringFilter by scope: system | org | private
curl "http://10.146.15.188:8080lt;server-address10.146.15.188:8080gt;/api/v1/rules?scope=private&enabled_only=true" \ -H "X-API-Key: dms_your_key"
POST /rules Create a new rule Auth required ▼
admin → any scope org owner → org or private user → private only
Request Body (JSON)
FieldTypeRequiredDescription
namestring✔Human-readable rule name
patternstring✔Python-compatible regex pattern
categorystringPII, Credential, Network, Finance, Health, Custom (default: custom)
flagsstringRegex flags: IGNORECASE, MULTILINE, DOTALL (comma-separated)
strategystringplaceholder (default) | asterisk | partial | hash
placeholderstringReplacement text, e.g. [EMAIL] (default: [MASKED])
weightintegerSensitivity 0–100; affects risk score (default: 5)
scopestringsystem | org | private (default: private)
enabledbooleanDefault: true
Example
{ "name": "SUSE Regcode", "pattern": "(?:Regcode-)[A-Z0-9]{5}", "category": "Identifier", "strategy": "placeholder", "placeholder": "[REGCODE]", "weight": 7, "scope": "org" }
PUT /rules/{rule_id} Update a rule Auth required ▼
admin → any rule org owner → own org rules user → own private rules

All fields are optional — send only what you want to change.

DELETE /rules/{rule_id} Delete a rule Auth required ▼
admin → any rule org owner → own org rules user → own private rules
PATCH /rules/{rule_id}/toggle Toggle rule enabled/disabled Auth required ▼

No body required. Flips the enabled flag.

PATCH /rules/{rule_id}/promote Change rule scope (promote / demote) Auth required ▼
Request Body
FieldRequiredDescription
scope✔Target scope: system | org
org_idRequired when scope is org
GET /rules-export Export all rules as JSON Auth required ▼
admin → all rulesorg owner → org rules only
POST /rules-import Import rules from JSON Auth required ▼
admin → system rulesorg owner → org rules
Request Body
{ "rules": [ /* array of rule objects */ ] }
POST /rules/fork-system Fork system rules into your org Auth required ▼
org owneradmin

Copies all current system rules into the caller's org as org-scoped rules. After forking, the org's rule set is independent of system rules — changes to system rules no longer affect the org. Idempotent: re-calling only copies rules not yet forked.

Use this when your org needs a customized base rule set diverging from the system defaults.

AI — Regex Generation
Generate masking regex patterns from natural language descriptions using a local LLM (Ollama / OpenCode).
GET /llm/models List available Ollama models Auth required ▼

Returns models installed in the local Ollama server. Returns 503 if Ollama is unreachable.

GET /llm/providers List supported LLM providers Auth required ▼

Returns ollama and opencode provider metadata including base URLs and default models.

POST /llm/generate-regex Generate a regex pattern using local LLM Auth required ▼
Request Body
FieldRequiredDescription
description✔Natural language description, e.g. "SUSE registration code like Regcode-XXXXX"
model✔Model name, e.g. qwen2.5-coder:7b
providerollama (default) | opencode
contextOptional extra hint, e.g. "format: s3://bucket/key"
Response 200
{ "pattern": "(?:Regcode-)[A-Z0-9]{5}", "flags": "", "placeholder": "[REGCODE]", "weight": 3, "description": "Matches SUSE registration codes in format Regcode-XXXXX", "suggested_name": "SUSE Regcode Pattern", "suggested_category": "Identifier", "examples": { "match": ["Regcode-ABC12", "Regcode-XYZ78"], "no_match": ["SUSE-ABC12", "ABC12"] }, "structured": true, "model": "qwen2.5-coder:7b", "provider": "ollama" }
The response fields map directly to POST /rules — you can pipe the output to create a rule immediately. Returns 503 if the LLM server is unreachable.
